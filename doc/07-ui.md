# 07 · 前端架构

> 状态：草案 · UI 是用户唯一接触的表面，paperclip 说"execution visibility without log worship"，我们要做对

## 设计原则（直接抄 paperclip，因为它对）

1. **Progressive disclosure** — 默认人类可读摘要，需要时再展开 raw log
2. **Output-first** — 工件、进度优先于 log 流
3. **Board-level abstraction** — 默认视图回答："谁在干什么、卡在哪、要不要批"
4. **Conversation 附在 work object 上** — 不要做通用 chat
5. **不要让用户理解 token / API key plumbing** — 配置入口最小化

## 实时数据策略

### SSE 通道

```typescript
// hooks/useSessionStream.ts
export function useSessionStream(sessionId: string) {
  const queryClient = useQueryClient()

  useEffect(() => {
    const es = new EventSource(`/api/v1/stream/sessions/${sessionId}`)

    es.addEventListener("event", (e) => {
      const event = JSON.parse(e.data)
      // 写入 query cache，触发组件更新
      queryClient.setQueryData(["events", sessionId], (old: Event[] = []) => {
        return [...old, event]
      })
    })

    es.addEventListener("status", (e) => {
      queryClient.invalidateQueries(["session", sessionId])
    })

    return () => es.close()
  }, [sessionId])
}
```

### 重连策略

EventSource 自带重连。但需要：

1. 重连后立即 fetch 错过的 events（用 last_event_id）
2. 网络断开时 UI 显示"disconnected"提示

### 全局 vs 局部

- 首页用全局通道 `/api/v1/stream/sessions`
- session 详情用局部通道 `/api/v1/stream/sessions/:id`
- 拓扑视图用全局通道（关心结构变化）

## 状态管理分层

| 状态 | 工具 | 例子 |
|------|------|------|
| Server state | TanStack Query | sessions、events、agents |
| Client state | Zustand | UI 偏好、选中项、过滤器 |
| Form state | react-hook-form | 编辑表单 |
| URL state | TanStack Router | filter / pagination 同步 URL |

**原则：** 能放 server state 就别放 client state。能同步 URL 就同步。

## 性能预算

| 指标 | 目标 |
|------|------|
| 首屏 TTI | < 1s |
| 切视图 | < 100ms |
| Bundle 大小（gzip） | < 300KB initial |
| 单 session 视图持续渲染 CPU | < 5% |

### 大量 events 怎么办

session 内可能有几千 events，不能一次渲染：

1. **虚拟列表** — react-virtuoso
2. **按阶段聚合** — 默认只展示 stage 摘要
3. **分页 fetch** — 滚动到底加载
